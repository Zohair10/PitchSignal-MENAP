## Project Overview

PitchSignal MENAP — a Next.js AI tool that helps Pakistan/MENAP founders understand why investors may reject them before they pitch. The hero output is region-specific investor objections generated by a LangChain.js pipeline running against Groq (LLM provider).

## Commands

```bash
npm run dev        # Start dev server (http://localhost:3000)
npm run build      # Production build (also runs TypeScript type checking)
npm run lint       # ESLint
npx tsc --noEmit   # Standalone type check (use ./node_modules/.bin/tsc if npx fails)
```

## Architecture

### Pages (App Router, no `src/` directory)

- `app/page.tsx` — Landing page
- `app/evaluate/page.tsx` — Founder input form (client component)
- `app/report/page.tsx` — Report display, reads encoded JSON from `?data=` URL param (client component)
- `app/api/evaluate/route.ts` — POST endpoint that runs the AI pipeline, returns fallback report on any error
- `app/api/extract/route.ts` — POST endpoint for PDF text extraction

### AI Pipeline (sequential, stateful)

All chains are in `lib/chains/`. They run sequentially, passing a `ReviewState` object between them:

```
POST /api/evaluate
  → validate input with Zod (FounderInputSchema)
  → runIntakeChain()          — extract startup profile from pitch text
  → injectKnowledgePack()     — inject regional data based on country
  → runStoryReviewChain()     — score narrative clarity
  → runMarketTractionChain()  — score regional market fit + traction credibility
  → runObjectionsChain()      — generate top 5 investor objections
  → runFinalMemoChain()       — assemble complete EvaluationReport
  → return report JSON (or PAYBRIDGE_FALLBACK_REPORT on error)
```

**Key design decision:** Chains do NOT use LangChain's `withStructuredOutput()` because Groq's `llama-3.3-70b-versatile` doesn't support `json_schema` response format. Instead, all chains use `callStructuredModel()` from `lib/services/structuredOutput.ts` which sends direct model messages, extracts JSON from the response text, and validates with Zod.

### LLM Provider

- Provider: **Groq** (OpenAI-compatible API)
- Config: `lib/services/model.ts` — uses `ChatOpenAI` from `@langchain/openai` with `baseURL: https://api.groq.com/openai/v1`
- Env vars: `GROQ_API_KEY`, `GROQ_MODEL` (default: `llama-3.3-70b-versatile`)

### Data Flow

- Report state lives in the URL only — no database. Reports are compressed with `lz-string` via `lib/utils/encodeReport.ts`
- The evaluate form POSTs to `/api/evaluate`, encodes the JSON response, and navigates to `/report?data=<encoded>`
- Fallback report (`lib/data/fallbackReport.ts`) is always returned on API failure

### Import Alias

`@/*` maps to project root (configured in `tsconfig.json`). E.g., `import { getModel } from "@/lib/services/model"`

## Key Schemas

All in `lib/schemas/`:
- `founderInput.schema.ts` — 14-field form input (8 required, 6 optional)
- `evaluationReport.schema.ts` — full report output with scores, objections, memo, fixes
- `chainOutputs.schema.ts` — intermediate chain outputs (intake, story, market, objections)
- `reviewState.schema.ts` — `ReviewState` interface passed between chains + `MENAPContext` type

## Regional Knowledge Pack

`lib/data/regionalKnowledgePack.ts` — the key differentiator. Contains Pakistan and MENAP-specific investor expectations, red flags, market stats. Injected into the pipeline after the intake chain. The report exposes `regionalSignalsUsed` to show judges this data powers the output.
