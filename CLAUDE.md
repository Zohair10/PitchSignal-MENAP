## Project Overview

PitchSignal MENAP ‚Äî a Next.js AI tool that helps Pakistan/MENAP founders understand why investors may reject them before they pitch. The pipeline auto-detects the startup's sector and applies sector-specific evaluation criteria. Hero output: region-specific, sector-aware investor objections generated by a LangChain.js pipeline running against Groq.

## Architecture

### Pages (App Router, no `src/` directory)

- `app/page.tsx` ‚Äî Landing page
- `app/evaluate/page.tsx` ‚Äî Founder input form (client component)
- `app/report/page.tsx` ‚Äî Report display, reads encoded JSON from `?data=` URL param (client component)
- `app/api/evaluate/route.ts` ‚Äî POST endpoint that runs the AI pipeline (sync), returns fallback report on any error
- `app/api/evaluate-stream/route.ts` ‚Äî POST endpoint with SSE streaming, shows real-time agent progress
- `app/api/extract-ocr/route.ts` ‚Äî POST endpoint for PDF OCR (receives client-rendered page images)

### AI Pipeline (sequential, stateful, sector-aware)

All chains are in `lib/chains/`. They run sequentially, passing a `ReviewState` object between them:

```
POST /api/evaluate or /api/evaluate-stream
  ‚Üí validate input with Zod (FounderInputSchema)
  ‚Üí runIntakeChain()          ‚Äî extract profile, detect sector (12 categories)
  ‚Üí injectSectorKnowledge()   ‚Äî load sector-specific criteria from sectorKnowledgePack
  ‚Üí injectKnowledgePack()     ‚Äî inject regional data based on country
  ‚Üí runStoryReviewChain()     ‚Äî score narrative clarity (sector-aware)
  ‚Üí runMarketTractionChain()  ‚Äî score regional market fit + traction (sector-specific rules)
  ‚Üí runObjectionsChain()      ‚Äî generate top 5 investor objections (sector + region specific)
  ‚Üí runFinalMemoChain()       ‚Äî assemble complete EvaluationReport
  ‚Üí return report JSON (or PAYBRIDGE_FALLBACK_REPORT on error)
```

**Key design decision:** Chains do NOT use LangChain's `withStructuredOutput()` because Groq's `llama-3.3-70b-versatile` doesn't support `json_schema` response format. Instead, all chains use `callStructuredModel()` from `lib/services/structuredOutput.ts` which sends direct model messages, extracts JSON from the response text, and validates with Zod.

### LLM Provider

- Provider: **Groq** (OpenAI-compatible API)
- Config: `lib/services/model.ts` ‚Äî uses `ChatOpenAI` from `@langchain/openai` with `baseURL: https://api.groq.com/openai/v1`
- Env vars: `GROQ_API_KEY`, `GROQ_MODEL` (default: `llama-3.3-70b-versatile`)

### Data Flow

- Report state lives in the URL only ‚Äî no database. Reports are compressed with `lz-string` via `lib/utils/encodeReport.ts`
- The evaluate form POSTs to `/api/evaluate-stream` (with `/api/evaluate` as fallback), encodes the JSON response, and navigates to `/report?data=<encoded>`
- Fallback report (`lib/data/fallbackReport.ts`) is always returned on API failure

### Import Alias

`@/*` maps to project root (configured in `tsconfig.json`). E.g., `import { getModel } from "@/lib/services/model"`

## Key Schemas

All in `lib/schemas/`:
- `founderInput.schema.ts` ‚Äî 3 required fields (startupName, country, pitchText), 11 optional fields extracted by AI
- `evaluationReport.schema.ts` ‚Äî full report output with `sectorCategory` (top-level), scores, objections, memo, fixes
- `chainOutputs.schema.ts` ‚Äî intermediate chain outputs (intake with sector enum, story, market, objections)
- `reviewState.schema.ts` ‚Äî `ReviewState` interface passed between chains, includes `detectedSector` and `sectorKnowledge`

## Sector System

### Sector Classification

Defined in `lib/data/sectorKnowledgePack.ts`. 12 categories: Fintech, SaaS, E-commerce, Healthtech, Edtech, Logistics, AgriTech, CleanTech/Energy, PropTech, AI/ML, D2C, Other.

The intake chain classifies into exactly one category via `z.enum(SECTOR_CATEGORIES)` ‚Äî the LLM must pick from the predefined list. Downstream chains receive `state.sectorKnowledge` with sector-specific red flags, investor expectations, key metrics, and MENAP nuances.

### Knowledge Packs

Two knowledge systems injected into the pipeline:
1. **Regional** (`lib/data/regionalKnowledgePack.ts`) ‚Äî Pakistan and MENAP investor expectations, market stats, red flags
2. **Sector** (`lib/data/sectorKnowledgePack.ts`) ‚Äî 12 sectors with evaluation lens, key metrics, red flags, investor expectations, regional nuances, bonus signals
